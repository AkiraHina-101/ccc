Clean release notes
===================

This folder is intended to be copied as one unit to the target machine.
Keep the folder structure unchanged:

  release_clean\
    app\
    main.py
    run.bat
    run_hidden.bat
    nastran_settings.json
    nastran_presets.json

Launchers
---------

Only Python 3.9 is supported in this release.

1. run.bat

   Normal launcher with console output. Use this first when checking a target
   machine because it prints Python, PYTHONPATH, and PySide2 import status.

2. run_hidden.bat

   Quiet launcher. Use this after run.bat works.

Library path
------------

This release does not include the Python/Qt library folder. The target machine
must already have the required libraries.

Edit both run.bat and run_hidden.bat if the target machine needs explicit paths:

  set "PYTHON_EXE=<TARGET_PYTHON39_DIR>\python.exe"
  set "EXTRA_LIBS=<TARGET_PYTHON_LIBS_DIR>"

For run_hidden.bat, PYTHON_EXE may point to pythonw.exe:

  set "PYTHON_EXE=<TARGET_PYTHON39_DIR>\pythonw.exe"

EXTRA_LIBS must point to a folder where Python 3.9 can import PySide2,
shiboken2, and any SSH-related packages used by this app.

App settings
------------

Edit nastran_settings.json or use the Settings screen in the app.

Values commonly changed per target machine/site:

  settings.server         SSH / Nastran server host
  settings.user           SSH user
  settings.password       SSH password, if password auth is used
  settings.ttmacro_path   TeraTerm macro executable/path, if required
  settings.win_prefix     Windows path prefix mapping, if required
  settings.versions       Nastran version list shown in the UI
  settings.queues         Queue list shown in the UI
  settings.ssh_buttons    SSH helper buttons/commands

Machine-specific data policy
----------------------------

Do not add personal user-profile paths into code files. If a target machine
needs an absolute path, put it in run.bat, run_hidden.bat, or configure it
through nastran_settings.json / the Settings screen.
